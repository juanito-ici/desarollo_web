Objetivos

Objetivos

  • Utilizar entorno de desarrollo visto en clase anterior
  • Utilizar herramientas colaborativas vistas en clase anterior (github)
  • Diseñar página HTML utilizando estilos y bootstrap (incluyendo su lógica de grillas)
  • Utilizar CSS para cambiar el aspecto de la página.
  • Utilizar Javascript para cambiar elementos de forma dinámica.

Desarrollo

Nivel noob: (nota máxima 4,0)

Utilizando su template favorito de Bootstrap inicie un proyecto desde cero, y personalícelo para su proyecto principal. La personalización debe incluir la adición de a lo menos:

  • Una imagen (no tiene que ser suya, pero debe ser distinta a las del template)
  • Un link a un sitio de interés
  • Un nuevo título
  • Un nuevo texto

Nivel meh ok: (nota máxima 5,5)

Organice los elementos anteriores utilizando el grid de Bootstrap, para que el diseño sea responsivo. Debe agregar por lo menos 6 elementos (pueden ser de ti po texto, imagen, video incrustado de Y ou T ube, mapas, etc. )

Agregue varios elementos de menú y verifique que el menú también sea responsivo.

Agregue un botón , que al apretarlo cambie mediante Javascript el estilo de uno de los texto s agregado s en el punto 2 .

Nivel pro: (nota máxima 7,0)

Modifique el estilo de alguno de los elementos anteriormente añadidos , agregando una nueva hoja de estilos, con un nuevo estilo. Esta modificación puede ser cambiar el tamaño de las letras, el color de fondo, agregar un borde a algún elemento, etc .

Modifique un estilo ya existe nte de su template de Bootstrap , indicando un comentario en el código que indique la modificación realizada .

Crear texto dinámico: e n la misma página d o nde agregó un text o plano en el punto 2 , añada un nuevo botón . Al hacer clic en este botón, dicho texto deberá cambiar por una entrada de datos (input) que contenga al texto en cuestión.

GIT

Pequeña ayuda con git:

  • git init
  • git add
  • git add .
  • git commit - m “comentario ”
  • git remote add origin
  • git push - u origi n master

Con estos pasos es suficiente para poder subir sus ar chivos a su repositorio remoto. E l paso 2 puede ser “ a ” o “ b ” , dependiendo de la situación.

Adicionalmente, en este enlace se explican claramente los comandos básicos de git: http://rogerdudler.github.io/git-guide/index.es.html

En caso de que sigan con dudas , cuando crean un repositorio remoto ,le da pasos a seguir para que sub a sus archivos a su repositorio , con el ejemplo de subir el archivo README.md.
